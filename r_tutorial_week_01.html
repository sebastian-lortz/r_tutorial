How to use this tutorial

Welcome to the R tutorial! You do not need any prior experience. Work through this page carefully, from top to bottom. Every short explanation is followed by a small piece of code to try and a Your turn task to do yourself.

General tips:

  • Read each part slowly and in order. Each step builds on the one before it.
  • Type the code yourself instead of copying and pasting it. You learn faster this way.
  • Errors are normal. Read the message and try again.
  • If you are stuck, ask a teacher.
  • Save your work often.

1 Your computer’s filing system

Before we start with RStudio, we discuss some important terms about your computer itself.

A file is a single object (a photo, a Word document, an R script). A folder (also called a directory) is a box that holds files and other folders. Folders can sit inside other folders, building a folder structure.

NoteExample: a tidy folder structure

A tidy folder structure can look like this:

Documents
└── first_year
    └── research_methods
        ├── data
        ├── scripts
        └── output

At the root of this structure is a Drive. Thus, a drive is the storage of your computer. On a personal computer you usually have only one drive. On top of the drive sit some common base folders, such as the Desktop (the screen you see when your computer starts), Documents (where files are usually stored), and Downloads (where files you download are saved).

It is important to keep this structure tidy. Work inside Documents, make one folder for your first-year studies. Inside that folder, make a folder for each course. Inside each course folder, you can sort your material further.

Do not build your folder structure inside the Downloads folder. Everything you download lands there automatically, so it quickly becomes messy.

Your turn

Create one folder for this whole course, so everything stays in one place.

  1. Open your file explorer (Windows Explorer on Windows, Finder on Mac).
  2. Go to your Documents folder.
  3. If it is not there already, create a folder for your studies and name it first_year.
  4. Inside first_year, create a new folder and name it research_methods.

Good practice for names:

  • Keep folder and file names simple: use lowercase letters, numbers, and underscores _ instead of spaces. For example, my data.csv can cause problems later, but my_data.csv does not.
  • Use short but clear names, so you know what is inside without opening the folder.
  • When a folder starts to hold many different things, make subfolders to group them.

You now have a home for your course work.

2 Opening RStudio

R is the engine that does the calculations. RStudio is the user interface you use to talk to that engine comfortably. You installed both before class (see the Brightspace instructions). We always open RStudio only and never open R itself.

Your turn

Open the RStudio program (not plain “R”).

ImportantStuck on installation?

Raise your hand right away. A teacher will help you.

2.1 The four panes

RStudio’s window is split into panes.

  • Editor (top-left): where you write and save code (empty for now).
  • Console (bottom-left): where R runs commands and shows results.
  • Environment (top-right): a list of everything you have stored (empty for now).
  • Files / Plots / Packages / Help (bottom-right): your file browser, graphs, installed packages, and help pages.

Find the Console. You will see a > symbol. That > means “R is ready and waiting for you.” If, while programming, this symbol is not shown, click in the console and press Esc (escape) on your keyboard to get the > symbol back.

4 R scripts

In this course we only work with R scripts, which are shown in the Editor pane. An R script is a place where you type and save lines of code, which are then run. A script runs from top to bottom.

We work with R scripts because your progress in RStudio is lost when you close the program. But if you save your R script, you can open it again next time and run each line to get back to where you stopped last time.

This is important for reproducibility: you can give your script to another person, and they get the same output. We never type our code straight into the console.

Your turn

Create a new R script: click File > New File > R Script.

In a script, we write notes with a # (hash) at the start of a line. A line that starts with # is called a comment. RStudio does not run a comment as code; it skips it. Comments help you and others understand what your code does.

Your turn

Click in the first line of your new R script. Type a # and then a title for your script, for example: # R tutorial - Week 1.

5 R as a calculator

The fastest way to meet R is to use it as a calculator.

Your turn

Type each line below into your R script. Press Enter after each one, so the lines sit underneath each other.

2 + 3
#> [1] 5
10 - 4
#> [1] 6
6 * 7
#> [1] 42
20 / 5
#> [1] 4

To run a line, click somewhere in that line and:

  • on Windows, hold Ctrl and press Enter;
  • on Mac, hold Cmd and press Enter.

R runs the line and jumps to the next one.

Your turn

Run the four calculations above, one by one.

R prints the answer back. Ignore the [1] in front for now – it just means “this is the first result.”

R also knows powers and square roots:

2 ^ 3    # 2 to the power of 3
#> [1] 8
sqrt(9)  # the square root of 9
#> [1] 3

Remember: anything after a # is a comment: a note for humans that R ignores. Use comments to remind yourself what code does.

6 Objects: storing values for later

Re-typing numbers every time is tedious. Instead we store a value in an object (sometimes called a variable) and give it a name. We do this with the assignment arrow <-, which you read as “gets.”

age <- 19

Read that as: “the object age gets the value 19.” Notice nothing was printed. The value is now stored. To see it, type the name on its own:

age
#> [1] 19
TipShortcut for the assignment arrow

Press Alt and - (minus) on Windows, or Option and - on Mac. RStudio types <- with the spaces for you.

Once stored, an object can be used in calculations just like the number itself:

self_efficacy <- 32   # a score on the self-efficacy scale we will collect
self_efficacy + 8
#> [1] 40

You can also overwrite an object by assigning a new value to the same name:

age <- 20   # age is now 20, the old value (19) is gone
age
#> [1] 20

Check the Environment pane (top-right): age and self_efficacy now appear there. That pane is your live list of stored objects.

Naming rules (important, R is strict):

  • Names may use letters, numbers, . and _, and must start with a letter.
  • No spaces. Use self_efficacy, not self efficacy.
  • Names are case-sensitive: Age, age, and AGE are three different objects.
Age   # we never created "Age" with a capital A, so R complains
#> Error: object 'Age' not found

That red message is an error, not a disaster. Here it simply means “I don’t know an object called Age.” Read errors as hints.

7 Operators

Operators are used to compare objects or numbers. They give back one or more logical values: either TRUE or FALSE.

These are the most common comparison operators:

Operator Meaning
== is equal to
!= is not equal to
> is greater than
< is smaller than
>= is greater than or equal to
<= is smaller than or equal to

For example:

10 > 5    # is 10 greater than 5?
#> [1] TRUE
10 < 5    # is 10 smaller than 5?
#> [1] FALSE
10 == 10  # is 10 equal to 10?
#> [1] TRUE
WarningWatch out

To ask “is it equal?” you must use two equals signs ==. A single = means something different and will not work here.

Your turn

In your script, check whether:

  1. 7 is greater than 3, and
  2. 4 is equal to 5.

Run each line and read the TRUE / FALSE answer.

8 Vectors: working with many values at once

In research we rarely have one number; we have many (one score per participant). A vector is an ordered collection of values. We build one with c(), which stands for combine. Put the values inside, separated by commas.

scores <- c(28, 35, 31, 40, 22)
scores
#> [1] 28 35 31 40 22

scores now holds five values in one object.

A powerful feature: arithmetic on a vector applies to every element at once. No loop needed.

scores + 10   # add 10 to each score
#> [1] 38 45 41 50 32
scores * 2    # double each score
#> [1] 56 70 62 80 44

Vectors do not have to be numbers. A character vector holds text (each piece in quotes), and a logical vector holds TRUE/FALSE values:

names  <- c("Sanne", "Tim", "Noor", "Mees", "Lotte")
passed <- c(TRUE, FALSE, TRUE, TRUE, FALSE)
names
#> [1] "Sanne" "Tim"   "Noor"  "Mees"  "Lotte"
passed
#> [1]  TRUE FALSE  TRUE  TRUE FALSE
WarningWatch out

Every value needs its own comma, and text needs quotes. c("Sanne" "Tim") (missing comma) gives an error; so does c(Sanne, Tim) (missing quotes – R would look for objects named Sanne and Tim).

Your turn

Create a vector engagement with the values 4, 5, 3, 5, 2 (imagine these are study-engagement ratings). Then print its mean() and its length().

9 Picking out values (indexing)

Often you want just one value, or a few, out of a vector. You do this with square brackets [ ] and a position number. R counts from 1, so the first element is position 1.

scores[1]   # the first score
#> [1] 28
scores[3]   # the third score
#> [1] 31

Pick several at once by putting a vector of positions inside:

scores[c(1, 2)]   # the first and second scores
#> [1] 28 35

A minus sign means “everything except”:

scores[-1]   # all scores except the first
#> [1] 35 31 40 22

Most powerful of all: put a condition inside the brackets to keep only the values that match. This is how you filter data.

scores > 30          # TRUE/FALSE for each score
#> [1] FALSE  TRUE  TRUE  TRUE FALSE
scores[scores > 30]  # keep only the scores above 30
#> [1] 35 31 40

Read the last line as: “give me the elements of scores where scores is greater than 30.”

TipGoing further (optional)

Combine conditions with & (“and”) or | (“or”), e.g. scores[scores > 25 & scores < 40].

Your turn

10 Functions

Functions have the form name(input). They take an input, apply operations, and give back an output. A function is a ready-made command: you write its name and put the input in round brackets. Many useful functions take a vector and summarise it:

length(scores)   # how many values?
#> [1] 5
sum(scores)      # add them all up
#> [1] 156
mean(scores)     # the average
#> [1] 31.2
min(scores)      # the smallest
#> [1] 22
max(scores)      # the largest
#> [1] 40

Your turn

Use the engagement vector you made earlier. Print its sum(), its max(), and its min().

11 Borrowing extra tools: packages and libraries

Base R already does a lot. Packages add extra ready-made tools written by others. Two steps, and people mix them up constantly:

  • Install a package: download it to your computer. You do this once.
  • Load a package with library(): switch it on. You do this every session you want to use it.
NoteAnalogy

Installing a package is like buying a book once and putting it on your shelf. library() is taking that book off the shelf each time you need it. Buying it again every day would be silly; so is re-installing.

The syntax (you do not need to run this today – we will install specific packages when the course needs them):

install.packages("psych")   # download once (note: name in quotes)
library(psych)              # switch on each session (no quotes here)
WarningWatch out

install.packages() needs the name in quotes; library() does not. And if you ever see Error: there is no package called ..., it means the package was never installed.

12 Saving your work

Save your R script by clicking the save symbol (the small floppy disk), or by pressing Ctrl and S on Windows (Cmd and S on Mac). You can also click File > Save.

The first time you save a new script, RStudio asks you where to put it. Choose your research_methods folder and give the file a clear name, for example week1.R.
